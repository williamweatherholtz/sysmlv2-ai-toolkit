# status-report

A VIEW skill (§3d). Generates the HTML project dashboard by invoking `report.py`.
The output is clearly marked as a derived, regenerable view — it is never committed
as authored content (per D0015, D0001).

## Sections of the Dashboard

1. **Header**: project name, generation timestamp, HEAD SHA
2. **Active Sprint**: current phase, estimated points, actual hours (if set),
   efficiency for this sprint
3. **Velocity Table**: all sprints with `estimatedPoints` set — shows estimated
   points, actual hours, efficiency (pts/h), and flags missing `actualHours`
4. **Trailing Averages**: 3-sprint and 5-sprint trailing velocity and efficiency
5. **Backlog Status**: counts (done / outstanding / suspect) from orient; simple
   names from the ready frontier (not raw SysML snippets)
6. **Recent Decisions**: last 10 decisions by D-number with their titles
7. **Recent Activity**: last 10 git commits (oneline)
8. **Generation footer**: "Generated by report.py — derived view, not authored content"

## How to Invoke

```powershell
# From repo root — generates and opens in browser:
python .engine\tools\report.py

# With explicit output path:
python .engine\tools\report.py --output docs\status.html

# Print to stdout:
python .engine\tools\report.py --stdout
```

The script requires only Python standard library + the `target\release\keel.exe`
binary for orient JSON. It does NOT require the conda/sysml kernel.

## Rules for This Skill

- **Never commit the generated HTML** unless saving to `docs/` as a tagged snapshot
  explicitly requested by the user, and clearly labelled with the generation date.
- **Never use a prior HTML report as authored truth.** Always regenerate from source.
- **If keel.exe is absent**, report.py falls back to direct tracking-file parsing for
  done/outstanding counts. The output will note the fallback.
- **If actualHours is missing** for a sprint, the efficiency cell is rendered as "—"
  and the trailing average skips that sprint.

## Maintenance

When new tracking sections are added (new sprint types, new schema fields), update
`report.py` to include them. The skill itself needs no update unless the report
structure changes.
